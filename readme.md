A prototype agent conductor system running on AWS.

#### Conductor?

As far as I know, an "Agent Orchestrator" largely refers to a system that spins up multiple agents to work on related goals, coordinating with one another using a whole slew of tools (e.g Agent Mail, beads, etc.).

The noun "conductor" is mostly unclaimed in this context, and I have it to mean that this system is largely responsible for the lifecycle of multiple agents working on *separate features in separate codebases*, rather than coordinating work between agents following a unified goal.
